'提交本地项目到github'

发表于 2017-07-20

提交本地项目到github

  1. 在github上新建项目，然后clone项目到本地，将要上传的项目放到此目录下
  2. 添加要上传的项目到暂存区：git add . (点表示上传所有)
  3. 提交：git commit -m ‘提交信息描述’
  4. 第一次提交执行命令：git push -u origin master (以后提交只需要git push origin master 即可)
  5. 新建分支：git checkout -b dev(分支名字)，此命令表示创建并切换，
    相当于以下两条命令：
    git branch dev
git checkout dev
    然后用 git branch 查看当前分支
  6. 切换回主分支： git checkout master
  7. 合并指定分支到当前分支：git merge dev
  8. 删除dev分支：git branch -d dev

搭建博客教程

发表于 2017-07-12

搭建博客教程


  1. 在github里创建project，名字和自己的名字一样（jojolfy.github.io）

  2. 初始化本地文件夹：hexo init 文件目录，然后进去到 cd 文件目录

  3. npm install hexo-deployer-git –save

  4. 在博客目录里生成静态文件：hexo g

  5. 更改_config.yml文件，里边的url改为自己的地址

  6. 启动服务测试：hexo s

  7. 更改_config.yml的type为git，并在后边添加repo属性，值为仓库地址

  8. 再次执行 hexo g 命令

  9. 最后上传 hexo d

vue中自定义指令

发表于 2017-06-24

全局指令

局部指令

指令定义函数提供了几个钩子函数（可选）：

  • bind: 只调用一次，指令第一次绑定到元素时调用，用这个钩子函数可以定义一个在绑定时执行一次的初始化动作。
  • inserted: 被绑定元素插入父节点时调用（父节点存在即可调用，不必存在于 document 中）。
  • update: 被绑定元素所在的模板更新时调用，而不论绑定值是否变化。通过比较更新前后的绑定值，可以忽略不必要的模板更新（详细的钩子函数参数见下）。
  • componentUpdated: 被绑定元素所在模板完成一次更新周期时调用。
  • unbind: 只调用一次， 指令与元素解绑时调用。

钩子函数的参数有：

  • el: 指令所绑定的元素，可以用来直接操作 DOM 。
  • binding: 一个对象，包含以下属性：
    • name: 指令名，不包括 v- 前缀。
    • value: 指令的绑定值， 例如： v-my-directive="1 + 1", value 的值是 2。
    • oldValue: 指令绑定的前一个值，仅在 update 和 componentUpdated 钩子中可用。无论值是否改变都可用。
    • expression: 绑定值的字符串形式。 例如 v-my-directive="1 + 1" ， expression 的值是 "1 + 1"。
    • arg: 传给指令的参数。例如 v-my-directive:foo， arg 的值是 "foo"。
    • modifiers: 一个包含修饰符的对象。 例如： v-my-directive.foo.bar, 修饰符对象 modifiers 的值是 { foo: true, bar: true }。
  • vnode: Vue 编译生成的虚拟节点。
  • oldVnode: 上一个虚拟节点，仅在 update 和 componentUpdated 钩子中可用。

自定义指令传参

  1. 指令中直接传data中的值：v-test="data"
  2. 指令中传一个函数：v-test="{fn:fn}"

指令原则上是针对DOM元素的 ，不存在this
应该尽可能的通过指令表达式把指令需要的内容传入
所以一般不推荐在指令内调用组件实例上的方法或获取组件实例的属性，当然一定要用的话,第三个参数vode的vnode.context就是Vue实例
